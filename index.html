Big O Notation Practice

Step One: Simplifying Expressions

  1. O(n + 10): O(n)
  2. O(100 * n): O(n)
  3. O(25): O(1)
  4. O(n^2 + n^3): O(n3)
  5. O(n + n + n + n): O(n)
  6. O(1000 * log(n) + n): O(n)
  7. O(1000 * n * log(n) + n): O(nlog(n))
  8. O(2^n + n^2): O(2n)
  9. O(5 + 3 + 1): O(1)
  10. O(n + n^(1/2) + n^2 + n * log(n)^10): O(n2)

Step Two: Calculating Time Complexity

  1. O(n)
  2. If n > 10 O(n), if n < 10 O(1)
  3. For n <= 10 O(n) For n > 10 O(1)
  4. O(n)
  5. O(n2)
  6. O(n)

Step Three: T/F & Short Answer

  1. True
  2. True
  3. False
  4. .indexOf array method has a complexity of O(n) because it iterates through an array one by one until it finds a match
  5. .includes array method has a complexity of O(n) because it iterates through an array one by one until it finds a match.
  6. .forEach array method has a complexity of O(n) because it iterates through an array and executes a provided function for each array element
  7. .sort array method has a complexity of O(nlog(n)) because it divides an array into smaller chunks, and inserts them using a merge sort
  8. .unshift array method has a complexity of O(n) because it shifts all existing elements to accomodate for the new element(s) at the beginning
  9. .push array method has a complexity of O(1) because it simply adds an element to an array and doesn't need to shift any other elements
  10. .splice array method has a complexity of O(n) because it has to shift existing elements to accomodate new element(s) or remove existing elements
  11. .pop array method has a complexity of O(1) because it involves updating an index of the last element when no argument is provided, but O(n) when an argument is given
  12. Object.keys() has a complexity of O(n) because it returns an array containing all the enumerable keys of the given object
